# FraudShield 项目安装部署

> **环境说明**：本指南以 **Ubuntu 22.04 LTS** ，配置 4C 8G。
>
> **项目定位**：多模态反诈智能体系统。

------

## 一、项目部署目标

将 FraudShield 系统完整部署到 Linux 服务器，实现以下运行形态：

- **前端**：使用 Vue 生产构建产物 `dist`，由 Nginx 直接托管。
- **后端**：使用 FastAPI + uvicorn 提供服务，监听 `8000` 端口。
- **安全**：Nginx 对外提供 HTTPS 访问。
- **代理**：Nginx 将 `/api/` 与 `/ws/` 转发到后端。
- **路由**：支持前端页面、静态资源及深层路由刷新访问。

------

## 六、从零开始部署的完整步骤

### 第 1-2 步：准备代码与恢复资源

按照上述第五节说明完成解压。

### 第 3 步：配置后端环境变量



```
cp /root/multimodal-fraud/backend/.env.expamle /root/multimodal-fraud/backend/.env
```

**关键配置项**：

- `SECRET_KEY` / `JWT_SECRET_KEY`
- `DATABASE_URL` (默认: `sqlite+aiosqlite:///./data/app.db`)
- `OPENAI_API_KEY` / `OPENAI_BASE_URL`
- `ZHIPU_AUDIO_API_KEY` / `VISION_API_KEY` 等

### 第 4-5 步：安装依赖

**后端：**

```
cd /root/multimodal-fraud/backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**前端：**



```
cd /root/multimodal-fraud/frontend
npm install
npm run build
# 构建后复制到静态目录
cp -a dist/. /var/www/fraud.songtf.cn/
```

### 第 6 步：启动后端服务

```
cd /root/multimodal-fraud/backend
source venv/bin/activate
python main.py
```

*健康检查：`curl http://127.0.0.1:8000/health` -> 应返回 `{"status":"ok"}`*

### 第 7-8 步：配置与重载 Nginx

**关键 Location 配置：**

```
# 前端静态托管
location / {
    root /var/www/fraud.songtf.cn;
    try_files $uri $uri/ /index.html;
}

# 后端 API/WS 转发
location /api/ { proxy_pass http://127.0.0.1:8000; }
location /ws/ { 
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

执行：`nginx -t && nginx -s reload`

## 八、运行状态说明

- **前端**：静态 `dist`，由 Nginx 托管。
- **后端**：FastAPI，由 Uvicorn 后台运行 (Port 8000)。
- **Nginx**：承担 HTTPS 卸载、静态资源分发、API/WS 转发。
