# 反诈前端视觉规范 v1

## 1. 目标与适用范围

本规范用于约束 `AFSO-020` 之后的整站 UI 改版，覆盖 Portal、Workbench 和专项分析页三条体验线。目标是在保留“反诈、预警、治理、证据”业务语义的前提下，彻底摆脱旧有 dark-tech 赛博舱视觉，建立一套更适合阅读、审计和工作流操作的新设计语言。

本规范只定义视觉与展示约束，不改变路由、请求、状态管理和组件业务职责。后续 token、全局基础层、布局和样板页任务都必须以本文件为引用依据。

## 2. 核心视觉语义

新的视觉方向统一为“证据卷宗 + 风险警戒”。

- 页面气质：克制、严肃、可审阅，不做娱乐化科技炫光。
- 信息表达：强调证据层级、状态对比和任务路径，而不是大面积背景特效。
- 场景区分：门户偏品牌可信度，后台偏治理工作台，专项分析偏工具流与结果呈现，但三者共享同一套语义颜色、边框和版式秩序。

## 3. 颜色系统

### 3.1 主色

- 主品牌色：深墨蓝 `#17304A`
- 主强调色：档案青 `#2F6B7A`
- 高亮点缀色：证据金 `#D6A85F`

使用规则：

- 主品牌色用于导航标识、标题强调、重点边框与主按钮正文框架。
- 档案青用于信息强调、筛选状态、图表中性主序列和说明型标签。
- 证据金只用于关键数据高亮、时间线节点、重要证据计数，避免满屏泛黄。

### 3.2 风险与状态色

- 危险色：警戒红 `#B94C3A`
- 警示底色：`#F6E4DE`
- 成功色：校验绿 `#4F7A5A`
- 信息色：中性蓝灰 `#5F7388`

使用规则：

- 危险色优先用于风险等级、异常预警、阻断动作和高优先级提示。
- 危险态要依赖颜色 + 文案/图标双重表达，不能只靠颜色区分。
- 成功色与信息色只能用于辅助反馈，不能抢占主 CTA 权重。

### 3.3 中性色

- 画布底色：`#F3EFE7`
- 内容底板：`#FCFAF6`
- 一级正文：`#1F2933`
- 二级正文：`#52606D`
- 弱说明：`#7B8794`
- 分隔线：`#D9D1C3`

要求：默认正文必须落在暖中性色系上，优先保障阅读对比度，不再以深色背景作为全站默认前提。

## 4. 背景与材质

- 页面背景采用“纸面/档案底”思路，可使用轻微纵向渐变或低对比纹理，但必须足够克制。
- 内容容器以实体浅底板为主，允许局部使用更深的标题条、状态条或目录栏建立节奏。
- 卡片优先表达为“卷宗卡 / 信息托盘”，而不是悬浮玻璃片。
- 模态框、下拉层、浮层必须有清晰边界和实体背景，避免背景内容穿透影响可读性。

## 5. 边框、圆角与阴影

- 主边框语言：1px 实线 + 低饱和暖灰描边。
- 强调边框：仅用于激活态、选中态、风险态，不用于所有组件默认常亮。
- 默认圆角：10px 到 18px，保持稳重，不使用过度未来感的大圆角胶囊作为通用语言。
- 阴影以短距离、低透明度为主，目标是分层，不是发光。

明确要求：

- 阴影不允许承担“品牌识别”职责。
- 组件层级优先靠版式、边框、留白和标题条区分，再考虑阴影。

## 6. 排版与字体

- 中文正文字体优先沿用系统可用的无衬线栈，确保构建稳定和中文覆盖完整。
- 标题允许使用更有重量感的字重和更紧的字距，但不额外引入高风险在线字体依赖。
- 标题层级建议控制在 3 级以内：页面标题、分区标题、卡片标题。
- 数据数字可使用更紧凑的数字风格，但必须与正文对比清晰。

排版要求：

- 正文行高保持在 `1.5` 到 `1.7`。
- 大段说明文本限制在可读宽度内，避免宽屏长行难以扫描。
- 英文标签、指标名和按钮文案以简洁为先，避免全大写赛博式命名。

## 7. 图标与插图

- 图标统一使用线性或轻量面性 SVG 图标，不使用 emoji。
- 风险、证据、告警、审核、档案等语义应建立稳定映射，避免同一语义在不同页面频繁换图形。
- 插图、背景装饰只允许作为低对比辅助，不能盖过核心数据与操作入口。

## 8. 动效与交互

- 页面与模块动画以 `opacity`、`transform` 为主，时长控制在 `160ms` 到 `280ms`。
- Hover 反馈优先使用边框、底色和阴影细微变化，避免放大抖动和强烈位移。
- 焦点态必须清晰可见，键盘导航不可被视觉重构破坏。
- 需要兼容 `prefers-reduced-motion`，减少非必要入场和背景动画。

## 9. 组件级视觉边界

### 9.1 布局骨架

- Portal 与 Workbench 应采用明确的页头、内容区、辅助信息区分层。
- 页头更接近“机构门户”与“治理台”而非悬浮 HUD。
- 导航、状态、筛选、搜索区域要有稳定节奏，避免层层嵌套发光容器。

### 9.2 卡片与分区

- 卡片分为证据卡、指标卡、信息卡、操作卡 4 类语义，视觉上共享底板和边框语言。
- 卡片头部可通过标题条、编号、状态标识建立差异，不依赖渐变底图堆特征。
- 空状态优先用说明文字、引导动作和轻量图形，不做大面积科幻插画。

### 9.3 按钮与表单

- 主按钮强调“执行动作”，样式实体、对比明确。
- 次按钮与幽灵按钮要保持弱化但仍可点击识别，不能因为去科技感而失去操作可见性。
- 输入框、选择器、Tabs、分页等基础控件以清晰边界和稳定焦点态为优先级。

## 10. 禁用清单

以下旧视觉元素默认禁止继续作为新主题方向：

1. 大面积冷蓝辉光和全局霓虹描边。
2. 以深黑蓝为全站默认画布的统一深色底。
3. 泛滥的玻璃拟态透明卡片和背景穿透层。
4. 依赖外发光、内发光作为选中态或品牌识别的按钮、卡片和导航。
5. 赛博网格、雷达扫描、HUD 面板边角作为通用容器背景。
6. 全大写、过度未来感的标题排版与按钮文案。
7. 悬浮位移过大、会打断阅读节奏的 hover 和入场动画。

如确需例外，必须在对应 issue 的 `notes` 中写明 `design_exception:<原因>`，并说明为什么不能复用本规范。

## 11. 后续任务引用规则

- `AFSO-030` 必须把新 token 命名对齐到“页面背景 / 内容底板 / 证据卡 / 风险标签 / 主按钮 / 说明 / 警示”等语义层，不再新增以 `tech` 或具体颜色命名的新 token。
- `AFSO-040` 必须让 Element Plus 与全局基础层落在本规范的中性色、边框、焦点和动效原则上。
- `AFSO-050` 与 `AFSO-060` 必须共用本规范的布局气质与组件边界，但允许在门户、后台、工具页之间保留内容密度差异。

## 12. 评审检查表

后续样式评审至少检查以下问题：

- 是否仍然以实体浅底板和清晰文本对比为默认前提。
- 是否重新引入了冷蓝辉光、透明玻璃、全站深色底等禁用元素。
- 是否把风险语义、主操作和说明信息清晰区分。
- 是否保持布局、卡片、按钮和表单的统一边界语言。
- 是否兼顾 hover、focus、disabled、empty 和 reduced-motion 场景。

## 13. 关联入口

- `src/main.ts` 负责全局样式注入入口。
- `src/styles/global.scss` 负责基础样式装配。
- `src/styles/tokens.scss` 与 `src/styles/theme-dark-tech.scss` 是后续 token 解耦与兼容映射的核心区域。
